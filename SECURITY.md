# 安全政策

ViaSix 会下载并执行第三方网络工具，也会在本机保存可能含连接凭据的 Mihomo YAML、Provider URL 和缓存。请负责任地报告安全问题。

## 支持范围

当前只维护主分支和最新发布版本。旧版本是否获得修复由维护者按漏洞影响单独评估；正式发布后应在此补充清晰的版本支持表。

## 报告漏洞

优先使用代码托管平台的私密漏洞报告功能。若仓库尚未启用该功能，请通过维护者提供的非公开联系方式报告，并在公开 Issue 中只说明“需要安全联系方式”，不要披露细节。

报告最好包含：

- 受影响版本、提交和 macOS / CPU 架构
- 可复现步骤、预期结果与实际结果
- 影响范围及可行的缓解措施
- 不含真实凭据的最小化测试样例

请不要公开提交 Mihomo YAML、旧 Xray 迁移样例中的真实凭据、UUID、服务器地址、订阅 URL、签名证书、Apple 公证凭据、下载令牌或用户数据。维护者确认修复和披露安排前，请避免公开漏洞细节。

## 供应链与运行组件

- 官方安装流程只从固定的上游 HTTPS 地址下载特定版本资产。
- 每个资产均使用 `RuntimeManifest.swift` 中固定的压缩包 SHA-256，并核对解压后 payload 的大小与 SHA-256。
- 更新组件时必须核对发布来源、版本、文件名、架构、压缩格式、双重校验值和许可证声明。
- 本地导入的组件由用户自行信任；ViaSix 不能证明其来源或完整性。

当前虚拟网卡模式不可启用。特权 helper 不得执行用户可写的 `Runtime/mihomo`，XPC 不得接受任意路径、argv、shell 或原始 YAML。任何真实 TUN 实现都必须使用固定签名的 root-owned 内核，并通过路由、DNS、进程身份和崩溃恢复审计。

若怀疑已提交过真实连接资料，应先轮换或吊销凭据，再清理当前分支和 Git 历史。仅从最新文件中删除并不能消除历史泄露。
