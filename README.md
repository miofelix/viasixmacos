# ViaSix for macOS

ViaSix 是一款原生 macOS Mihomo 客户端与网络工具，用于管理代理配置、代理组、活动连接和路由规则，也能测试 IPv4 / IPv6 边缘节点并把优选结果应用到自己的配置。常用状态和控制集中在 Clash 风格的信息架构中，也可以通过菜单栏快速操作。

> [!IMPORTANT]
> ViaSix 不提供代理账号、订阅、服务器或网络接入服务。使用“规则”或“全局”模式前，你需要准备自己有权使用的兼容服务器配置；“直连”模式不需要服务器配置或候选节点。

> [!NOTE]
> 当前仓库提供源码构建和本地 ad-hoc 打包流程。普通用户应只安装维护者明确发布、签名并公证的产物；在正式下载地址公布前，请按下文从源码构建。

## 主要功能

- 使用内置 IPv4 / IPv6 地址列表进行节点测速
- 支持自定义地址文件、单个 IP 和 CIDR 网段
- 按延迟、丢包率、下载速度和地区筛选候选节点
- 展示前三名节点与完整测速结果
- 一键应用节点；本地代理运行时可自动重新连接
- 在“首页”单独复测当前节点，并显示清晰的测速状态
- 通过 Mihomo Controller 实时查看并切换代理组
- 使用带鉴权的 Mihomo WebSocket 监控连接，断线后自动重连
- 在代理页切换规则、全局或直连模式，并批量测试组内节点延迟
- 查看 Proxy / Rule Provider 状态、订阅用量并触发在线更新
- 查看上传、下载、累计流量、活动连接与本次运行的关闭历史；支持排序、详情检查和终止连接
- 展示实时流量趋势与 Mihomo 内存占用
- 检查内核实际加载的路由规则
- 独立管理、导入和编辑 Mihomo 配置档
- 提供规则、全局和直连三种代理模式
- 启动、停止和重启本地 HTTP / SOCKS 代理
- 可选启用 macOS 系统代理，并在本地代理停止或应用退出时恢复原设置
- 可选启用 TUN 虚拟网卡模式：由固定签名 Mihomo 管理自动路由、DNS 劫持、IPv4/IPv6 与回环防护
- 在设置中独立安装/修复 TUN 后台服务与特权 Mihomo；服务未获批准时保持不可用
- 检测直连或代理状态下的出口 IP
- 展示出口 IP 的国家、地区、城市、网络运营商、ASN 与时区
- 通过菜单栏控制常用操作
- 在本机保存设置、地址列表和代理配置，不收集遥测

ViaSix 使用第三方项目 XIU2/CloudflareSpeedTest 完成节点测速，使用 Mihomo 提供本地代理能力。CloudflareSpeedTest 并非 Cloudflare 官方产品。组件在首次使用时按需从各自上游 Release 下载，也可以从本机导入。

## 系统要求

- macOS 14 Sonoma 或更高版本
- Apple Silicon（arm64）或 Intel（x86_64）Mac
- 安装组件、节点测速和出口检测时需要网络连接

## 安装

### 使用发布包

1. 从维护者公布的正式发布页获取适合当前 Mac 的 `ViaSix.app`。
2. 将应用拖入“应用程序”文件夹。
3. 打开 ViaSix。

如果 macOS 提示无法验证本地构建，请先确认应用来源可信，再在 Finder 中按住 Control 点击应用并选择“打开”。不要为了运行 ViaSix 关闭 Gatekeeper 或系统完整性保护。

### 从源码构建

开发环境需要 Xcode 16.3 / Swift 6.1 或更高版本；Xcode 16.3 本身要求 macOS 15.2 或更高版本。

```bash
make check
make app
open dist/ViaSix.app
```

本地 `make app` 生成 ad-hoc 签名包，只适合开发和冒烟测试。完整说明见[开发文档](Docs/DEVELOPMENT.md)。

## 快速开始

1. 打开“设置 → 运行组件”，按需分别安装 CloudflareSpeedTest 与 Mihomo，或为对应组件导入自定义可执行文件。
2. 进入“测速”，选择 IPv6、IPv4、自定义文件或 IP / CIDR。
3. 点击“开始测速”，查看候选节点；单独测速不需要代理配置。
4. 在“首页”或“设置 → 本机代理”中选择代理模式：“规则”“全局”或“直连”。
5. 使用“规则”或“全局”时，在“配置”中导入或编辑内联节点、Proxy Provider、代理组和规则；内联节点可以按需应用测速结果。使用“直连”时可以跳过这一步。
6. 如需修改监听地址、端口、UDP 或网络接入方式，打开“设置 → 本机代理”。需要 TUN 时先在“设置 → 虚拟网卡服务”安装并批准服务，再选择“虚拟网卡”。
7. 回到“首页”启动本地代理。
8. 可以启用“系统代理”，让遵循 macOS 代理设置的应用自动使用本地端点；也可以使用 TUN 接管不支持系统代理的应用流量。系统代理与 TUN 互斥。
9. 代理运行后，可在“代理”“连接”和“规则”中操作 Mihomo 的实时状态。

ViaSix 使用 `profile.yaml`、本机代理设置和当前模式生成 Mihomo 运行配置。同一个回环端口可作为 HTTP 或 SOCKS5 代理使用。“规则”模式默认让私有地址直连、其余流量按配置规则处理；“全局”模式让所有进入本地代理的流量通过代理；“直连”模式让这些流量直接连接。服务器表单支持 VLESS、VMess、Trojan 和 Shadowsocks，也可以粘贴对应分享链接；多节点、Proxy Provider、代理组和规则可以使用高级 YAML 编辑器。

系统代理是可选的网络接入方式，只影响遵循 macOS 代理设置的应用。TUN 模式由签名约束的 LaunchDaemon 启动固定 Mihomo，自动管理路由、DNS 劫持和回环防护；它与系统代理互斥。ViaSix 会在停止或退出时先恢复系统代理、停止当前登录用户自己启动的 TUN 会话，再允许退出；其他用户的会话只显示脱敏占用状态。

更完整的配置格式、参数说明、备份方法和排错步骤见 [用户指南](Docs/USER_GUIDE.md)。

## 数据与隐私

ViaSix 的可变数据默认保存在：

```text
~/Library/Application Support/ViaSix/
```

- 设置、测速地址列表和代理配置只保存在本机。
- ViaSix 不收集遥测，也不要求创建 ViaSix 账号。
- 安装某个组件时只会连接该组件上游项目的固定版本发布地址。
- 节点测速会访问测速组件的默认地址，或你填写的自定义测速 URL。
- 出口 IP 检测可选择自动、IPv4 或 IPv6；自动模式默认使用 `https://api.myip.la/cn?json`，也可在“设置 → 应用与数据”中改为其他 HTTP / HTTPS 服务，强制地址族时使用 IP.SB 的对应专用端点。检测成功后，ViaSix 会通过 `https://ipwho.is/<IP>?lang=zh-CN` 补充中文国家、地区、城市、邮编、网络运营商、ASN 和时区；地理信息服务不可用时仍会显示已检测到的 IP。
- 本地代理只允许监听本机回环地址和端口，不会主动向局域网开放端口。
- Mihomo Controller 只监听 `127.0.0.1`，并使用保存在私有数据目录中的随机密钥鉴权。

第三方组件及网络服务受各自许可证、隐私政策和可用性约束。
完整的数据、网络连接、保留和删除说明见[隐私说明](PRIVACY.md)。

## 安全说明

- 代理配置可能包含 UUID、域名、密钥或其他敏感资料，请勿上传、公开或发送给不受信任的人。
- ViaSix 不会把第三方可执行文件直接存放在源码仓库中；在线安装会从组件上游发布页下载并校验文件完整性。
- ViaSix 只管理自己启动的测速和代理进程，不会按进程名结束其他应用。
- 关闭主窗口不会退出 ViaSix；应用会继续驻留菜单栏。需要停止所有自有进程时，请从菜单栏选择“退出 ViaSix”。
- TUN 使用 app bundle 内固定签名、root-only 安装的 Mihomo；用户自定义可执行文件永远不会进入特权 helper。后台服务、会话 journal、配置和进程身份均经过校验。
- 启用系统代理时会通过 macOS 原生网络配置接口修改当前启用的网络服务，系统可能要求授权；TUN 模式不会同时启用系统代理。

## 常见问题

### ViaSix 是否自带代理服务？

不自带。ViaSix 负责节点测速和本地代理进程管理，但不提供服务器、账号或订阅。你必须使用自己有权访问的 Mihomo 配置、分享链接或连接参数。

### 为什么首次启动本地代理时提示连接尚未配置？

“规则”和“全局”模式需要在“配置”中填写自己的连接参数、读取分享链接，或在 Mihomo YAML 编辑器中导入配置。Provider 配置可以直接提供节点；如果只需要保留本地端点并直接连接，可以切换到“直连”模式，无需服务器配置或候选节点。

### ViaSix 会修改系统代理吗？

仅在你启用“系统代理”后才会修改。ViaSix 会在本地代理运行期间，把当前启用的 macOS 网络服务的 HTTP、HTTPS 和 SOCKS 代理指向本地端点；停止本地代理或退出 ViaSix 时恢复原设置。未启用时，可以在浏览器、下载工具、终端或其他目标应用中单独设置本地端点。

### 系统代理能接管全部网络流量吗？

系统代理只影响遵循 macOS 代理设置的应用。若需要接管不遵循系统代理的应用，请在“设置 → 虚拟网卡服务”完成服务批准和特权 Mihomo 安装后选择“虚拟网卡”。TUN 会接管符合 Mihomo 路由规则的 IPv4/IPv6 流量，但 VPN、睡眠唤醒或其他网络工具改变路由后，ViaSix 可能要求恢复或重新启动会话。

### 关闭窗口后为什么菜单栏仍有 ViaSix？

这是正常行为。ViaSix 可以在菜单栏继续测速或维持本地代理。选择菜单栏中的“退出 ViaSix”才会完整退出。

### 测速结果是否代表实际代理速度？

不一定。测速结果反映候选边缘 IP 在当前网络中的表现；最终体验还会受到你的代理服务器、传输配置、网络拥塞和目标站点影响。

### 如何备份或迁移设置？

完全退出 ViaSix 后，备份 `~/Library/Application Support/ViaSix/`。详细的文件用途和恢复步骤见 [用户指南](Docs/USER_GUIDE.md)。
