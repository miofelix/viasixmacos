# ViaSix 用户指南

本指南面向使用 ViaSix 的 macOS 用户，介绍首次配置、节点测速、本地代理、数据备份和常见问题。

> [!IMPORTANT]
> ViaSix 不提供代理账号、订阅、服务器或网络接入服务。你需要自行准备一份有权使用的兼容 Xray JSON 配置。ViaSix 也不能判断第三方代理服务是否合法、可信或可用。

## 1. 认识 ViaSix

ViaSix 包含两个相互配合的功能：

- **节点测速**：测试 IPv4 / IPv6 候选地址的延迟、丢包和下载速度。
- **本地代理**：把选中的节点写入你的 Xray 配置，并在本机 `127.0.0.1:11451` 提供 HTTP / SOCKS5 代理。

测速快并不意味着代理一定可用。你的账号、端口、传输方式、TLS、Host、路径等连接资料仍来自你自己的 Xray 配置。

## 2. 首次配置

### 2.1 安装运行组件

1. 打开 ViaSix。
2. 进入“设置”。
3. 选择“安装官方组件”。
4. 等待 CloudflareSpeedTest 和 Xray-core 均显示为已就绪。

ViaSix 会根据当前 Mac 的处理器架构下载对应版本，并在安装前校验文件完整性。该操作需要访问 GitHub Releases。

如果你已经在本机准备了组件，也可以使用“导入本地组件”。选择解压后的目录或相关文件时，ViaSix 会识别：

- `cfst`
- `xray`
- `geoip.dat`
- `geosite.dat`

建议导入完整的 Xray 发布目录，以免缺少数据文件。

### 2.2 准备自己的 Xray JSON

ViaSix 自带的模板只包含安全占位符，不能直接连接网络。你可以：

1. 在“设置”的“代理配置”区域选择“导入 Xray JSON…”。
2. 选择你自己的 `.json` 文件。
3. 导入成功后再进行节点测速和代理启动。

更换配置前应先停止本地代理。ViaSix 会校验 JSON 的基本结构，但不会验证账号是否有效，也不会替你购买或生成连接资料。

如果你选择手动编辑，可在“设置”中打开代理配置文件。保存前建议保留一份备份。

### 2.3 兼容配置要求

导入的是完整 Xray JSON，而不是 `vless://` 链接或订阅地址。配置至少需要满足以下约定：

- 所有入站都必须显式绑定 `127.0.0.1`、`::1` 或 `localhost`，不能监听 `0.0.0.0` 或其他局域网地址。
- 包含监听 `127.0.0.1:11451` 的 mixed 入站，否则 ViaSix 无法确认本地代理已经就绪。
- `outbounds` 中存在一个 `tag` 为 `proxy` 的出站。
- 该出站包含非空的 `settings.vnext` 数组。
- `settings.vnext[0]` 中包含你自己的端口和用户资料。
- TLS、WebSocket、gRPC 或其他传输参数必须与你自己的服务端一致。

结构示意：

```json
{
  "inbounds": [
    {
      "listen": "127.0.0.1",
      "port": 11451,
      "protocol": "mixed",
      "settings": {
        "auth": "noauth",
        "udp": true
      }
    }
  ],
  "outbounds": [
    {
      "tag": "proxy",
      "protocol": "vless",
      "settings": {
        "vnext": [
          {
            "address": "将由 ViaSix 应用所选节点",
            "port": 443,
            "users": [
              {
                "id": "你自己的 UUID",
                "encryption": "none"
              }
            ]
          }
        ]
      }
    }
  ]
}
```

此示例只说明 ViaSix 识别的结构，不是一份可直接使用的配置。

ViaSix 在应用节点时只会修改 `proxy` 出站中第一个 `vnext.address`。端口、用户、TLS Server Name、Host、路径和其他配置保持不变。因此，导入的连接必须本来就适合通过候选边缘 IP 访问。

> [!CAUTION]
> Xray 配置通常包含敏感连接资料，并以普通文件形式保存在本机。不要把配置、截图或备份提交到公开仓库。

## 3. 运行节点测速

### 3.1 选择地址来源

进入“节点优选”，选择以下来源之一：

- **IPv6**：使用 ViaSix 内置的 IPv6 地址列表。
- **IPv4**：使用 ViaSix 内置的 IPv4 地址列表。
- **自定义文件**：选择纯文本或 CSV 地址列表。
- **自定义 CIDR**：直接输入单个 IP、CIDR 或使用英文逗号分隔的组合。

自定义地址示例：

```text
2606:4700::/32, 104.16.0.0/12
```

只测试你有权访问且符合用途的地址范围。较大的 CIDR、过高的线程数或“测试全部 IPv4”会明显增加耗时和网络负载。

### 3.2 选择测速方式

- **TCPing**：速度快、资源占用较低，适合先筛选可达性和延迟。
- **HTTPing**：可获得 HTTP 状态和地区信息，适合需要地区筛选的场景。

多数用户可以先使用默认参数。需要更精确控制时，可调整：

- 测速端口
- 延迟上下限
- 丢包率上限
- 下载速度下限
- 延迟测速线程数
- 单 IP Ping 次数
- 下载测速节点数量与时长
- 自定义测速 URL
- 地区代码过滤

参数会自动保存在本机。选择“重置”可恢复默认测速参数。

### 3.3 开始、停止和查看结果

1. 确认地址来源和参数。
2. 点击“开始测速”。
3. 等待进度完成；需要中止时点击“停止”。
4. 查看“最快节点”和完整候选列表。
5. 点击节点卡片或结果行以应用该地址。

ViaSix 会在每次测速前清理旧结果，因此失败的任务不会显示成之前的成功结果。

如果本地代理正在运行，切换节点后 ViaSix 会重新启动代理。切换期间连接可能短暂中断。

## 4. 使用本地代理

### 4.1 启动和检查

1. 确认已经安装 Xray 组件。
2. 确认已经导入自己的代理配置。
3. 至少选择一个候选节点。
4. 回到“总览”，点击“启动本地代理”。
5. 状态显示运行中后，在目标应用中设置代理。

默认地址：

| 类型 | 主机 | 端口 |
| --- | --- | --- |
| HTTP / HTTPS 代理 | `127.0.0.1` | `11451` |
| SOCKS5 代理 | `127.0.0.1` | `11451` |

ViaSix 使用 mixed 入站，因此两种协议共用同一个端口。

### 4.2 在应用中设置

不同应用的设置名称可能是“代理服务器”“HTTP Proxy”“SOCKS5”或“网络代理”。将服务器填写为 `127.0.0.1`，端口填写为 `11451`。

ViaSix 不会修改 macOS 系统代理。只需要在希望使用代理的应用中配置即可。

### 4.3 在终端中使用

仅为当前终端会话设置：

```bash
export HTTP_PROXY=http://127.0.0.1:11451
export HTTPS_PROXY=http://127.0.0.1:11451
export ALL_PROXY=socks5h://127.0.0.1:11451
```

使用 `socks5h` 时，支持该写法的程序会通过代理解析域名。关闭终端窗口或执行以下命令可清除这些变量：

```bash
unset HTTP_PROXY HTTPS_PROXY ALL_PROXY
```

也可以单次测试：

```bash
curl --proxy http://127.0.0.1:11451 https://example.com
```

### 4.4 检测出口 IP

在“总览”中选择“检测”：

- 本地代理未运行时，显示当前直连出口。
- 本地代理运行时，通过本地代理请求并显示出口信息。

检测服务不可用、网络受限或代理配置错误时，检测可能失败，但这不一定代表节点测速失败。

### 4.5 关闭窗口与退出

关闭主窗口不会退出 ViaSix。本地代理和菜单栏入口仍会保留。

需要完全退出时，从菜单栏选择“退出 ViaSix”。ViaSix 会停止自己启动的测速和代理进程，然后保存设置。

## 5. 自定义组件路径

高级用户可以在“设置”中指定 CFST 或 Xray 可执行文件。留空时，ViaSix 按以下顺序查找：

1. 设置中指定的自定义路径
2. ViaSix 管理的组件
3. Homebrew 常用目录
4. 当前进程的 `PATH`

自定义文件必须与当前 Mac 架构兼容并具有执行权限。自定义 Xray 还需要能找到与其版本匹配的 `geoip.dat` 和 `geosite.dat`。

如无特殊需要，建议使用 ViaSix 管理的官方组件，减少路径和版本不匹配问题。

## 6. 应用数据、备份与恢复

数据目录：

```text
~/Library/Application Support/ViaSix/
```

主要内容：

| 路径 | 用途 |
| --- | --- |
| `Data/preferences.json` | 测速参数、组件路径和当前节点 |
| `Data/ip.txt` | IPv4 地址列表 |
| `Data/ipv6.txt` | IPv6 地址列表 |
| `Data/template.json` | 你的代理连接模板，可能包含敏感资料 |
| `Data/config.json` | 按当前节点生成的运行配置 |
| `Data/result.csv` | 最近一次测速结果 |
| `Runtime/` | ViaSix 管理的运行组件 |

`config.json` 是根据 `template.json` 和当前节点生成的文件。需要修改连接资料时，应编辑或重新导入 `template.json`，不要把 `config.json` 当作长期配置来源。

界面中的运行记录只保留当前会话，退出应用后会清空。

### 6.1 备份

1. 停止测速和本地代理。
2. 从菜单栏退出 ViaSix。
3. 在 Finder 中选择“前往”>“前往文件夹”。
4. 输入 `~/Library/Application Support/`。
5. 复制整个 `ViaSix` 文件夹到安全位置。

备份可能包含代理凭据，应使用受信任的存储位置。

### 6.2 恢复

1. 完全退出 ViaSix。
2. 先备份当前 `ViaSix` 数据文件夹。
3. 将备份内容恢复到 `~/Library/Application Support/ViaSix/`。
4. 重新打开应用并检查组件、代理配置和当前节点。

跨版本恢复后如果组件不兼容，可以在设置中重新安装官方组件。ViaSix 会保留用户修改过的地址列表和代理模板。

### 6.3 恢复为全新状态

完全退出应用后，把现有 `ViaSix` 数据文件夹移动到其他位置，再重新打开应用。ViaSix 会创建新的默认数据。确认不再需要旧数据前，不要直接删除原文件夹。

新的默认代理模板不包含可用账号，重置后需要重新导入自己的配置。

## 7. 排查问题

### 运行组件未就绪

- 确认网络可以访问 GitHub Releases。
- 重新选择“安装官方组件”。
- 如果使用本地导入，确认文件完整、架构正确且可执行。
- 在“运行记录”中查看具体下载、解压或校验错误。

### 提示“代理连接尚未配置”

- 导入自己的 Xray JSON。
- 确认没有保留 `00000000-0000-0000-0000-000000000000` 或 `example.com` 等占位值。
- 确认 `outbounds` 中有 `tag: "proxy"` 的出站。
- 确认 `settings.vnext` 非空。

### 导入代理配置失败

- 确认文件是有效 JSON，而不是订阅链接、二维码文本或 `vless://` URI。
- 确认代理已经停止。
- 按“兼容配置要求”检查 `proxy` 出站结构。
- 查看“运行记录”中的详细错误。

### 本地代理启动超时

- 确认导入配置包含监听 `127.0.0.1:11451` 的 mixed 入站。
- 确认端口没有被其他应用占用。
- 检查 Xray 配置中的端口、UUID、TLS、Server Name、Host 和路径。
- 确认已经选择节点。

可在终端中只读检查端口占用：

```bash
lsof -nP -iTCP:11451 -sTCP:LISTEN
```

不要在不清楚进程用途时强制结束它。

### 没有测速结果

- 尝试恢复默认筛选参数。
- 检查地址文件或 CIDR 是否有效。
- 暂时降低下载速度下限或放宽延迟、丢包限制。
- 检查自定义测速 URL 是否可以访问。
- 尝试在 TCPing 和 HTTPing 之间切换。

### 代理已启动但目标应用无法联网

- 确认目标应用代理类型、主机和端口填写正确。
- 使用“出口 IP”检测判断代理请求是否成功。
- 检查自己的 Xray 账号和传输参数。
- 尝试其他候选节点。
- 查看“运行记录”中的 Xray 输出。

### 出口 IP 检测失败

出口检测依赖第三方服务 `api.myip.la`。服务不可用、DNS 或网络策略受限时可能失败。可稍后重试，并结合目标应用的实际联网结果判断。

## 8. 安全建议

- 只使用自己有权访问的服务器和账号。
- 不要公开 `template.json`、日志中可能出现的连接信息或完整数据备份。
- 不要导入来源不明的 Xray 配置或可执行文件。
- 正式使用优先选择签名并经过 Apple 公证的 ViaSix 发布包。
- 发现配置泄露后，应立即在服务端撤销或更换相应凭据。

第三方组件及网络服务信息见 [第三方声明](../THIRD_PARTY_NOTICES.md)。
