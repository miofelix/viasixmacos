# ViaSix 用户指南

本指南面向使用 ViaSix 的 macOS 用户，介绍首次配置、节点测速、本地代理、数据备份和常见问题。

> [!IMPORTANT]
> ViaSix 不提供代理账号、订阅、服务器或网络接入服务。使用“规则”或“全局”模式时，你需要自行准备有权使用的兼容服务器配置；“直连”模式不需要服务器配置或候选节点。ViaSix 也不能判断第三方代理服务是否合法、可信或可用。

## 1. 认识 ViaSix

ViaSix 包含两个相互配合的功能：

- **节点测速**：测试 IPv4 / IPv6 候选地址的延迟、丢包和下载速度。
- **本地代理**：运行 Xray，在仅限本机访问的回环端点提供 HTTP / SOCKS5 代理，并按“规则”“全局”或“直连”模式处理进入该端点的连接。

“规则”和“全局”模式会把选中的节点写入生成的 Xray 运行配置。测速快并不意味着代理一定可用；账号、端口、传输方式、TLS、Host、路径等连接资料仍来自你自己的服务器配置。“直连”模式不使用服务器或候选节点，但仍需要 Xray 组件来提供本地端点。

## 2. 首次配置

### 2.1 安装运行组件

1. 打开 ViaSix。
2. 进入“设置 → 运行组件”。
3. 选择“安装组件”。
4. 等待 CloudflareSpeedTest 和 Xray-core 均显示为已就绪。

ViaSix 会查询上游最新正式版本，根据当前 Mac 的处理器架构下载对应资产，并使用 Release 提供的 SHA-256 在安装前校验文件完整性。该操作需要访问 GitHub API 与 GitHub Releases。

CloudflareSpeedTest 是 XIU2 维护的独立第三方项目，并非 Cloudflare 官方产品。这里的“上游组件”表示从 CloudflareSpeedTest 与 Xray-core 各自项目的正式 Release 获取。

如果你已经在本机准备了组件，也可以使用“导入组件”。选择解压后的目录或相关文件时，ViaSix 会识别：

- `cfst`
- `xray`
- `geoip.dat`
- `geosite.dat`

建议导入完整的 Xray 发布目录，以免缺少数据文件。

如果只使用节点测速，到这里即可进入“节点”。启动本地代理需要 Xray 组件；只有“规则”和“全局”模式需要服务器配置与候选节点，“直连”模式不需要。

### 2.2 配置代理模式、服务器与本机代理

ViaSix 自带的服务器配置只包含安全占位符，不能用于“规则”或“全局”模式。VLESS、VMess、Trojan 和 Shadowsocks 连接可以直接使用表单：

1. 在“首页”或“设置 → 本机代理”中选择“规则”“全局”或“直连”。
2. 使用“规则”或“全局”时，打开“设置 → 服务器连接”。选择“手动配置”填写协议、服务器地址、端口、认证信息、传输方式和 TLS / REALITY 参数，或选择“分享链接”读取 VLESS、VMess、Trojan 或 Shadowsocks 链接。
3. 在“设置 → 本机代理”中点击“编辑”，按需修改监听地址、监听端口、UDP、协议嗅探、私有地址直连、日志级别，以及是否随本地代理启用系统代理。
4. 保存后再进行节点测速和代理启动；“直连”模式可以直接启动，不要求服务器配置或已选节点。

更换配置前应先停止本地代理。ViaSix 会校验结构和必填项，但不会验证账号是否有效，也不会替你购买或生成连接资料。

应用启动时会提前检查当前模式所需的资料。“规则”和“全局”模式缺少有效服务器配置时会显示为“尚未就绪”，代理启动入口保持禁用，并提供前往设置填写参数的提示；“直连”模式只要本机配置有效即可启动。

可视化表单未覆盖的 Xray 出站参数可以在“设置 → 服务器连接 → 高级”中编辑服务器 JSON。该编辑器管理 `proxy` 出站，本机监听参数仍在“设置 → 本机代理”中填写。也可以从同一“高级”菜单导入完整 Xray JSON，ViaSix 会读取其中的服务器出站和本机 mixed 入站。

### 2.3 兼容配置要求

“设置 → 服务器连接 → 高级”可以导入完整 Xray JSON；分享链接请通过“设置 → 服务器连接 → 分享链接”读取。完整 JSON 至少需要满足以下约定：

- 所有入站都必须显式绑定回环地址（IPv4 的 `127.0.0.0/8`、`::1` 或 `localhost`），不能监听 `0.0.0.0` 或其他局域网地址。
- 包含一个端口在 1–65535 范围内的回环 `mixed` 入站，否则 ViaSix 无法确认本地代理已经就绪。回环 IPv4 可使用 `127.0.0.0/8`，也可使用 `::1` 或 `localhost`；端口不要求固定值。
- “规则”和“全局”模式要求 `outbounds` 中存在一个 `tag` 为 `proxy` 的出站。
- 该 `proxy` 出站包含非空的 `settings.vnext`（VLESS / VMess）或 `settings.servers`（Trojan / Shadowsocks）数组。
- 对应数组的第一个服务器对象中包含你自己的端口和用户资料；“直连”模式可以不包含 `proxy` 出站。
- TLS、WebSocket、gRPC 或其他传输参数必须与你自己的服务端一致。

结构示意：

```json
{
  "inbounds": [
    {
      "listen": "127.0.0.1",
      "port": 11451,
      "protocol": "mixed",
      "settings": {
        "auth": "noauth",
        "udp": true
      }
    }
  ],
  "outbounds": [
    {
      "tag": "proxy",
      "protocol": "vless",
      "settings": {
        "vnext": [
          {
            "address": "将由 ViaSix 应用所选节点",
            "port": 443,
            "users": [
              {
                "id": "你自己的 UUID",
                "encryption": "none"
              }
            ]
          }
        ]
      }
    }
  ]
}
```

此示例只说明 ViaSix 识别的结构，不是一份可直接使用的配置。

在“规则”或“全局”模式中，ViaSix 应用节点时只会修改 `proxy` 出站中第一个服务器对象的 `address`。端口、用户、TLS Server Name、Host、路径和其他配置保持不变。因此，导入的连接必须本来就适合通过候选边缘 IP 访问。“直连”模式生成运行配置时不会使用 `proxy` 出站。

> [!CAUTION]
> Xray 配置通常包含敏感连接资料，并以普通文件形式保存在本机。不要把配置、截图或备份提交到公开仓库。

## 3. 运行节点测速

### 3.1 选择地址来源

进入“节点”，选择以下来源之一：

- **IPv6**：使用 ViaSix 内置的 IPv6 地址列表。
- **IPv4**：使用 ViaSix 内置的 IPv4 地址列表。
- **自定义文件**：选择纯文本或 CSV 地址列表。
- **自定义 CIDR**：直接输入单个 IP、CIDR 或使用英文逗号分隔的组合。

自定义地址示例：

```text
2606:4700::/32, 104.16.0.0/12
```

只测试你有权访问且符合用途的地址范围。较大的 CIDR、过高的线程数或“测试全部 IPv4”会明显增加耗时和网络负载。

### 3.2 选择测速方式

- **TCPing**：速度快、资源占用较低，适合先筛选可达性和延迟。
- **HTTPing**：可获得 HTTP 状态和地区信息，适合需要地区筛选的场景。

多数用户可以先使用默认参数。需要更精确控制时，可调整：

- 测速端口
- 延迟上下限
- 丢包率上限
- 下载速度下限
- 延迟测速线程数
- 单 IP Ping 次数
- 下载测速节点数量与时长
- 自定义测速 URL
- 地区代码过滤

参数会自动保存在本机。选择“恢复默认设置…”可恢复默认测速参数。

### 3.3 开始、停止和查看结果

1. 确认地址来源和参数。
2. 点击“开始测速”。
3. 等待进度完成；需要中止时点击“停止”。
4. 查看候选列表，点击结果行只会预选节点。
5. 确认后点击“应用节点”；代理运行时会先提示是否重新连接。

测速进行中和失败后，页面会保留上次成功结果供比较，但不会允许把旧结果应用到代理。

在“首页”点击“测试当前节点”可以只复测已选节点。它会沿用当前协议、端口、URL、线程和下载设置，但不会让批量测速的延迟、丢包、速度或地区筛选条件把该节点过滤掉。开始新的复测时，旧的当前节点结果会先清除；取消或返回不匹配结果时也不会回写旧数据。

如果本地代理正在运行，切换节点后 ViaSix 会重新启动代理。切换期间连接可能短暂中断。

## 4. 使用本地代理

### 4.1 启动和检查

1. 确认已经安装 Xray 组件。
2. 在“首页”中选择“规则”“全局”或“直连”模式。
3. 使用“规则”或“全局”时，确认已经配置服务器并选择候选节点；使用“直连”时可以跳过服务器与节点配置。
4. 按需打开“系统代理”；也可以保持关闭，稍后在目标应用中手动配置本地端点。
5. 打开“本地代理”开关。
6. 状态显示运行中后检查目标应用的网络连接。

应用会在“首页”、“设置 → 本机代理”和菜单栏中显示从配置自动读取的端点。内置示例默认使用以下地址：

| 类型 | 主机 | 端口 |
| --- | --- | --- |
| HTTP / HTTPS 代理 | `127.0.0.1` | `11451` |
| SOCKS5 代理 | `127.0.0.1` | `11451` |

ViaSix 使用 mixed 入站，因此两种协议共用同一个端口。在“设置 → 本机代理”中修改回环主机或端口后，保存并重新启动代理即可应用。

### 4.2 选择代理模式

三种模式只决定进入 ViaSix 本地代理的连接如何出站：

| 模式 | 出站行为 | 启动要求 |
| --- | --- | --- |
| 规则 | 默认让私有地址直连，其余流量通过服务器 | 有效服务器配置和已选节点 |
| 全局 | 所有进入本地代理的 TCP / UDP 流量都通过服务器 | 有效服务器配置和已选节点 |
| 直连 | 所有进入本地代理的 TCP / UDP 流量都直接连接 | 仅需 Xray 与有效本机配置 |

“规则”模式当前使用内置基础规则。在“设置 → 本机代理”的编辑面板中关闭“私有地址直连”后，私有地址也会按默认代理出站处理；当前版本不提供 Clash 规则订阅、规则集或按应用分流。

代理模式与网络接入方式相互独立。不开启系统代理时，只有手动指向本地端点的应用会进入上述路由；开启系统代理后，遵循 macOS 代理设置的应用会进入同一本地端点。在“直连”模式下启用系统代理，连接仍会经过本地 Xray 端点，但最终直接出站。

### 4.3 使用系统代理

在“首页”或“设置 → 本机代理”中打开“系统代理”后：

- 如果本地代理已经运行，ViaSix 会立即尝试启用系统代理；否则会在 Xray 启动成功后启用。
- ViaSix 会把当前启用的 macOS 网络服务的 HTTP、HTTPS 和 SOCKS 代理指向本地 mixed 端点。
- 如果网络服务原来启用了 PAC，ViaSix 会在本次会话中暂时停用 PAC，并在恢复时还原原配置。
- 停止本地代理或退出 ViaSix 时，会先恢复启用前保存的 macOS 代理设置；应用异常中断后，下次启动也会尝试完成恢复。
- 如果其他应用在 ViaSix 运行期间修改了某个网络服务，ViaSix 不会用旧快照覆盖该服务的新设置，并会在“日志”中记录提示。

系统代理不等同于虚拟网卡模式，只影响遵循 macOS 代理设置的应用。忽略系统代理的应用、部分命令行工具和其他系统流量不会自动进入 ViaSix；当前版本不会通过 TUN 接管全部流量。

### 4.4 在应用中手动设置

不同应用的设置名称可能是“代理服务器”“HTTP Proxy”“SOCKS5”或“网络代理”。将服务器和端口填写为“设置 → 本机代理”里显示的本地端点。

不启用系统代理时，只需要在希望使用 ViaSix 的应用中配置即可。即使启用了系统代理，不遵循 macOS 代理设置的应用仍可能需要单独配置。

### 4.5 在终端中使用

以下示例使用内置模板的默认端点；如果你修改了本机配置，请替换为“设置 → 本机代理”里显示的端点：

```bash
export HTTP_PROXY=http://127.0.0.1:11451
export HTTPS_PROXY=http://127.0.0.1:11451
export ALL_PROXY=socks5h://127.0.0.1:11451
```

使用 `socks5h` 时，支持该写法的程序会通过代理解析域名。关闭终端窗口或执行以下命令可清除这些变量：

```bash
unset HTTP_PROXY HTTPS_PROXY ALL_PROXY
```

也可以单次测试：

```bash
curl --proxy http://127.0.0.1:11451 https://example.com
```

### 4.6 检测出口 IP

在“首页”中选择“检测”：

- 本地代理未运行时，显示当前直连出口。
- 本地代理运行时，通过本地代理请求并显示出口信息。
- “自动”由当前网络和检测服务选择可用的地址族，并在结果旁明确显示 IPv4 或 IPv6。
- “IPv4”或“IPv6”只请求对应地址族；如果当前网络或代理不支持该地址族，检测会明确失败，不会回退到另一种地址族。

检测服务不可用、网络受限或代理配置错误时，检测可能失败，但这不一定代表节点测速失败。

自动模式默认使用 `https://api.myip.la/cn?json`。如果需要使用自建服务或其他兼容 API，可在“设置 → 应用与数据”中填写 HTTP / HTTPS 地址。服务应返回带有 `ip` 字段的 JSON，或直接返回一个 IP 文本；响应内容会经过 IP 格式校验。

强制 IPv4 和 IPv6 模式分别使用 `https://api-ipv4.ip.sb/ip` 与 `https://api-ipv6.ip.sb/ip`。ViaSix 还会校验服务返回的地址族，避免把回退结果误标为所选类型。自定义检测服务只用于自动模式。

无论选择“自动”、IPv4 还是 IPv6，主检测成功后都会使用同一网络路径请求 IPWho 的地理信息接口 `https://ipwho.is/<IP>?lang=zh-CN`。结果会分为两层显示：国家、地区、城市和邮编，以及网络运营商（或 ISP）、ASN 和时区。地理信息请求失败、字段缺失或返回地址与主检测结果不一致时，不会覆盖已经检测到的 IP，只是不显示相应的补充字段。

### 4.7 关闭窗口与退出

关闭主窗口不会退出 ViaSix。本地代理和菜单栏入口仍会保留。

需要完全退出时，从菜单栏选择“退出 ViaSix”。ViaSix 会恢复本次会话应用的系统代理设置，停止自己启动的测速和代理进程，然后保存设置。关闭主窗口本身不会触发恢复，因为应用仍在运行。

## 5. 自定义组件路径

高级用户可以在“设置 → 运行组件”中指定 CFST 或 Xray 可执行文件。留空时，ViaSix 按以下顺序查找：

1. “设置 → 运行组件”中指定的自定义路径
2. ViaSix 管理的组件
3. Homebrew 常用目录
4. 当前进程的 `PATH`

自定义文件必须与当前 Mac 架构兼容并具有执行权限。自定义 Xray 还需要能找到与其版本匹配的 `geoip.dat` 和 `geosite.dat`。

如无特殊需要，建议使用 ViaSix 管理的上游组件，减少路径和版本不匹配问题。

## 6. 应用数据、备份与恢复

数据目录：

```text
~/Library/Application Support/ViaSix/
```

主要内容：

| 路径 | 用途 |
| --- | --- |
| `Data/preferences.json` | 测速参数、组件路径和当前节点 |
| `Data/ip.txt` | IPv4 地址列表 |
| `Data/ipv6.txt` | IPv6 地址列表 |
| `Data/server.json` | 服务器 `proxy` 出站，可能包含 UUID、域名和密钥 |
| `Data/local-proxy.json` | 本机监听、协议行为、代理模式和系统代理偏好 |
| `Data/template.json` | 完整 Xray 配置兼容文件，可能包含敏感资料 |
| `Data/config.json` | 按当前模式生成的 Xray 运行配置 |
| `Data/system-proxy.json` | 系统代理启用期间的 macOS 设置恢复快照，恢复完成后删除 |
| `Data/result.csv` | 最近一次测速结果 |
| `Runtime/` | ViaSix 管理的运行组件 |

`Data/local-proxy.json` 使用以下字段：

| 字段 | 用途 |
| --- | --- |
| `listenAddress` | 本地 mixed 入站监听地址，只允许回环地址 |
| `port` | 本地 HTTP / SOCKS 共用端口 |
| `udpEnabled` | 是否允许 mixed 入站处理 UDP |
| `sniffingEnabled` | 是否启用 Xray 协议嗅探 |
| `bypassPrivateNetworks` | 规则模式下是否让私有地址直连 |
| `logLevel` | Xray 日志级别 |
| `routingMode` | `rule`、`global` 或 `direct` |
| `systemProxyEnabled` | 是否在本地代理运行期间启用 macOS 系统代理 |

旧版本文件缺少 `routingMode` 或 `systemProxyEnabled` 时，ViaSix 分别按 `rule` 和 `false` 读取。

`config.json` 是派生文件：“规则”和“全局”模式根据服务器配置、本机代理设置及当前节点生成；“直连”模式只需要本机设置。不要直接编辑 `config.json`；服务器参数在“设置 → 服务器连接”中管理，本机行为在“设置 → 本机代理”中管理。`system-proxy.json` 只用于恢复操作系统设置，也不应手动修改。

“日志”中的记录只保留当前会话，退出应用后会清空。

### 6.1 备份

1. 停止测速和本地代理。
2. 从菜单栏退出 ViaSix。
3. 在 Finder 中选择“前往”>“前往文件夹”。
4. 输入 `~/Library/Application Support/`。
5. 复制整个 `ViaSix` 文件夹到安全位置。

备份可能包含代理凭据，应使用受信任的存储位置。

### 6.2 恢复

1. 完全退出 ViaSix。
2. 先备份当前 `ViaSix` 数据文件夹。
3. 将备份内容恢复到 `~/Library/Application Support/ViaSix/`。
4. 重新打开应用并检查组件、代理配置和当前节点。

跨版本恢复后如果组件不兼容，可以在“设置 → 运行组件”中重新安装组件。ViaSix 会保留用户修改过的地址列表和代理模板。

### 6.3 恢复为全新状态

完全退出应用后，把现有 `ViaSix` 数据文件夹移动到其他位置，再重新打开应用。ViaSix 会创建新的默认数据。确认不再需要旧数据前，不要直接删除原文件夹。

新的默认代理模板不包含可用账号，重置后需要重新导入自己的配置。

### 6.4 升级应用

1. 完全退出 ViaSix。
2. 备份 Application Support 中的 `ViaSix` 文件夹。
3. 用新版本 `.app` 替换旧版本。
4. 启动后检查运行组件、当前节点和代理配置。

ViaSix 只迁移与历史默认内容完全匹配的资源；用户编辑过的列表和模板会保留。跨较大版本升级时，请同时阅读发布说明和[变更日志](../CHANGELOG.md)。

### 6.5 卸载

- 只删除应用：退出 ViaSix 后移除 `ViaSix.app`。Application Support 中的数据仍会保留。
- 彻底卸载：退出应用并删除 `ViaSix.app`，再删除 `~/Library/Application Support/ViaSix/`。

彻底删除前请确认不再需要代理配置或备份。Time Machine、云盘等备份中的副本需按对应产品规则另行清理。

## 7. 排查问题

### 运行组件未就绪

- 确认网络可以访问 GitHub Releases。
- 在“设置 → 运行组件”中重新选择“安装组件”。
- 如果使用本地导入，确认文件完整、架构正确且可执行。
- 在“日志”中查看具体下载、解压或校验错误。

### 提示“代理连接尚未配置”

- 该提示只影响“规则”和“全局”模式；如果只需要直接出站，可以切换到“直连”。
- 使用服务器模式时，在“设置 → 服务器连接”中导入自己的 Xray JSON、读取分享链接或手动填写连接资料。
- 确认没有保留 `00000000-0000-0000-0000-000000000000` 或 `example.com` 等占位值。
- 确认 `outbounds` 中有 `tag: "proxy"` 的出站。
- 确认 `settings.vnext` 或 `settings.servers` 非空。

### 导入代理配置失败

- 文件导入只接受有效的完整 Xray JSON；`vless://`、`vmess://`、`trojan://` 和 `ss://` 请通过“设置 → 服务器连接 → 分享链接”读取。
- 确认代理已经停止。
- 按“兼容配置要求”检查 `proxy` 出站结构。
- 查看“日志”中的详细错误。

### 本地代理启动超时

- 确认“设置 → 本机代理”中的监听端点正确，并且端口没有被其他应用占用。
- 使用“规则”或“全局”时，检查 Xray 配置中的端口、UUID、TLS、Server Name、Host 和路径，并确认已经选择节点。
- 使用“直连”时，确认 Xray 组件可用且本机监听设置有效。

可在终端中只读检查端口占用：

```bash
lsof -nP -iTCP:11451 -sTCP:LISTEN
```

不要在不清楚进程用途时强制结束它。

### 没有测速结果

- 尝试恢复默认筛选参数。
- 检查地址文件或 CIDR 是否有效。
- 暂时降低下载速度下限或放宽延迟、丢包限制。
- 检查自定义测速 URL 是否可以访问。
- 尝试在 TCPing 和 HTTPing 之间切换。

### 代理已启动但目标应用无法联网

- 未启用系统代理时，确认目标应用代理类型、主机和端口填写正确。
- 已启用系统代理时，确认状态显示为“已启用”，并确认目标应用遵循 macOS 代理设置。
- 使用“出口 IP”检测判断代理请求是否成功。
- 使用“规则”或“全局”时，检查自己的 Xray 账号和传输参数，并尝试其他候选节点。
- 检查当前代理模式是否符合预期；“直连”不会使用服务器。
- 查看“日志”中的 Xray 输出。

### 系统代理启用或恢复失败

- 确认本地代理已经成功运行，并查看“日志”中的 macOS 网络设置错误。
- 如果系统要求授权，请完成授权后重试。
- 停止本地代理时如提示无法恢复，ViaSix 会尽量保留本地监听，避免系统应用指向已经关闭的端口；不要强制结束进程，先重试关闭系统代理或本地代理。
- 应用异常退出后重新打开 ViaSix，它会尝试使用保存的快照恢复上次设置。
- 如果某个网络服务已被其他应用修改，ViaSix 会保留其当前值；必要时在 macOS“系统设置 → 网络”中核对该服务的代理配置。

### 出口 IP 检测失败

自动出口检测默认依赖第三方服务 `api.myip.la`，也可以在“设置 → 应用与数据”中改用其他服务；强制 IPv4 / IPv6 检测依赖 IP.SB 的对应端点，位置补全依赖 `ipwho.is`。服务不可用、DNS、所选地址族或网络策略受限时可能失败。可稍后重试、切换地址族，并结合目标应用的实际联网结果判断。

## 8. 安全建议

- 只使用自己有权访问的服务器和账号。
- 不要公开 `server.json`、`template.json`、日志中可能出现的连接信息或完整数据备份。
- 不要导入来源不明的 Xray 配置或可执行文件。
- 正式使用优先选择签名并经过 Apple 公证的 ViaSix 发布包。
- 发现配置泄露后，应立即在服务端撤销或更换相应凭据。

第三方组件及网络服务信息见 [第三方声明](../THIRD_PARTY_NOTICES.md)。
本机数据和网络连接边界见[隐私说明](../PRIVACY.md)，安全问题报告方式见[安全政策](../SECURITY.md)。
