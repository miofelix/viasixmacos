# ViaSix 用户指南

ViaSix 的主要用途是让 Mac 流量通过一个经过优选的 IPv6 代理入口。它不提供服务器、账号或订阅，你需要使用自己有权访问的 Mihomo 配置。

## 1. 理解 IPv6 链路

规则或全局模式下的链路是：

```text
Mac 应用流量
  → 系统代理和/或 TUN
  → ViaSix 本地 Mihomo
  → 所选 IPv6 地址
  → 远程代理
  → 目标网站
```

ViaSix 保证 Mac 到远程代理入口使用所选 IPv6。远程代理访问网站时可能使用 IPv4 或 IPv6，所以最终出口不一定是 IPv6。

应用包含五个主工作区：

- **首页**：启停连接，查看实时流量，选择代理模式，独立控制系统代理和虚拟网卡；
- **IPv6 优选**：测试、比较并应用 IPv6 节点；
- **连接配置**：导入、查看和编辑 Mihomo YAML；
- **日志**：查看代理、测速、TUN 和应用事件；
- **设置**：管理运行组件、TUN 服务、本机参数和应用数据。

## 2. 首页控制

### 2.1 流量统计

连接运行后，首页会显示：

- 近约 10 分钟的上下行速率曲线；
- 当前上传 / 下载速率；
- Mihomo 进程内存占用。

数据来自本机 Mihomo Controller 的 WebSocket 推送。停止连接后曲线与速率会清空。菜单栏在运行中会以两行短格式显示上下行速率，便于不打开主窗口时查看。

### 2.2 代理模式

- **规则**：私有、回环和链路本地地址直连，其余流量走当前 IPv6 主代理；
- **全局**：所有进入 Mihomo 的代理流量统一走当前 IPv6 主代理；
- **直连**：不加载远程代理，运行配置只包含 `MATCH,DIRECT`。

规则和全局需要有效 IPv6 节点及可注入地址的内联代理。直连不需要节点或连接配置。

### 2.3 网络设置

“系统代理”和“虚拟网卡模式”是两个独立开关：

- **系统代理**：在连接运行期间配置 macOS HTTP、HTTPS 和 SOCKS 代理，停止或退出时恢复；
- **虚拟网卡模式**：通过 TUN 接管不遵循系统代理的应用流量；
- **同时开启**：TUN 与系统代理共同运行；
- **同时关闭**：ViaSix 只提供本地 mixed 端口，供手动配置的应用使用。

切换 TUN 时，ViaSix 会串行停止当前运行方式、保存偏好并按新方式重新启动。系统代理偏好不会被 TUN 切换改写。

## 3. 首次配置

### 3.1 安装运行组件

打开“设置 → 运行组件”：

- 安装 CloudflareSpeedTest 以执行 IPv6 优选；
- 如果准备关闭 TUN、使用用户态本地 Mihomo，请同时安装 Mihomo。

### 3.2 准备 TUN

如需虚拟网卡，在“设置 → 虚拟网卡服务”完成：

1. 安装或批准系统服务；
2. 安装或修复特权 Mihomo；
3. 确认服务、运行时和功能集均已就绪。

本地 ad-hoc 构建首次安装时会请求管理员授权。日常启动、停止和重连不需要重复授权。

TUN 网络栈、MTU 和严格路由仍在“设置 → 本机代理”中配置。通常保留 Mixed、1500 和默认严格路由设置即可。

### 3.3 导入连接配置

打开“连接配置”，导入 `.yaml` 或 `.yml`。配置必须包含至少一个可替换 `server` 的 VLESS、VMess、Trojan 或 Shadowsocks 内联节点。

```yaml
proxies:
  - name: My VLESS
    type: vless
    server: origin.example.com
    port: 443
    uuid: 11111111-1111-1111-1111-111111111111
    network: ws
    tls: true
    servername: origin.example.com
    ws-opts:
      path: /proxy
      headers:
        Host: origin.example.com
```

ViaSix 只替换主代理的服务器地址，不改动端口、凭据、传输、TLS/REALITY、SNI、Host 或路径。

Provider-only 配置不能运行。导入 YAML 中的代理组、Provider、规则 Provider、子规则和自定义规则不会进入实际运行配置。

Cloudflare Pages 生成配置可以省略 `server`，但必须声明：

```yaml
x-viasix:
  version: 1
  primary-server: selected-ip
```

导入这种配置前，需要先应用当前 IPv6 节点。`x-viasix` 不会改变首页代理模式、系统代理、TUN 或日志设置。

## 4. IPv6 优选

1. 打开“IPv6 优选”；
2. 使用内置 IPv6 列表、自定义文件或输入 IPv6/CIDR；
3. 调整 TCPing/HTTPing、端口、筛选条件和性能参数；
4. 开始测速；
5. 按延迟、丢包率和速度比较结果；
6. 应用一个 IPv6 地址。

ViaSix 不显示内置 IPv4 数据源，也不会应用 IPv4 结果。如果连接正在运行，应用新节点后会重新连接。

## 5. 启动与检查

首页链路卡会显示：

- 当前网络接入组合；
- 当前 IPv6 节点；
- 配置是否支持注入 IPv6；
- 连接是否运行。

选择规则或全局时，缺少 IPv6、缺少内联代理或 TUN 未就绪都会阻止启动并给出修复提示。选择直连时会跳过 IPv6 和代理配置检查，但如果请求了 TUN，TUN 服务仍必须就绪。

出口 IP 卡片显示最终网站出口，并不证明客户端入口地址族。判断 IPv6 入口是否生效，应查看首页当前节点和日志。

## 6. 日志

日志是长期保留的独立界面，可查看：

- 应用启动和配置同步；
- IPv6 节点应用与重连；
- 系统代理启用、恢复和回滚；
- TUN 服务、会话和恢复；
- Mihomo 启停和错误输出；
- CloudflareSpeedTest 进度与失败原因；
- 出口检测结果。

界面支持按来源和级别筛选、跟随最新记录、改变排序方向和清空当前会话日志。清空只影响内存记录，不修改配置或测速结果。

## 7. 数据、备份与恢复

默认数据目录：

```text
~/Library/Application Support/ViaSix/
```

主要文件：

- `Data/preferences.json`：节点、测速和应用偏好；
- `Data/profile.yaml`：用户连接配置，可能包含敏感凭据；
- `Data/local-proxy.json`：代理模式、网络开关、本地监听和 TUN 参数；
- `Data/Mihomo/config.yaml`：可重新生成的用户态运行配置；
- `Data/result.csv`：最近一次成功测速结果；
- `Logs/`：组件日志目录。

本版本不会迁移旧 Xray 配置，也不会恢复旧兼容模式。备份和恢复前应完全退出 ViaSix，不要在应用运行时覆盖文件。

## 8. 常见问题排查

### 提示“需要先选择有效的 IPv6 节点”

打开“IPv6 优选”并应用一个 IPv6 地址。IPv4 结果不能成为当前节点。

### 提示“当前配置无法注入 IPv6 节点”

当前 YAML 只有 Provider，或没有可替换 `server` 的内联代理。请改用内联配置。

### 提示“需要先准备 TUN 服务”

进入“设置 → 虚拟网卡服务”，检查系统批准、特权 Mihomo 和 helper 功能；或者在首页关闭虚拟网卡，改用用户态 Mihomo。

### 最终出口是 IPv4

这是允许的。ViaSix 保证本机到代理入口使用 IPv6，不保证远程代理到目标网站的地址族。

### 与 VPN 或其他网络工具冲突

先停止 ViaSix 和其他 TUN/VPN 工具，再单独启动。必要时调整 TUN 网络栈或严格路由，并查看日志中的路由、DNS 和恢复信息。

## 9. 安全建议

- 只使用自己有权访问的代理配置和测速目标；
- 不要公开 `profile.yaml` 或包含凭据的日志片段；
- 优先使用维护者签名并公证的正式应用；
- 不要关闭 Gatekeeper 或系统完整性保护来绕过安装问题；
- 完全退出应用后再删除数据目录或恢复备份。
